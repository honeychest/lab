## [2단계] T4-TASK.md 검수

### 준비
- `/devcontext/dev-cycle/T4-TASK.md` 읽기

### 체크 1 — T3-ARCH ↔ T4-TASK 대응
- [ ] T3-ARCH의 각 처리 흐름(REQ별)이 T4-TASK의 TASK로 1:1 대응되는가? → 대응 없는 항목 발견 시 **[높음]**
- [ ] T3-ARCH에서 명세된 모든 REST API 엔드포인트가 T4-TASK에 구현 TASK로 존재하는가? → 없으면 **[높음]**
- [ ] T3-ARCH에서 명세된 모든 WS 채널이 T4-TASK에 구현 TASK로 존재하는가? → 없으면 **[높음]**
- [ ] T3-ARCH에서 명세된 모든 DB 테이블/컬럼 변경이 T4-TASK에 구현 TASK로 존재하는가? → 없으면 **[높음]**
- [ ] T3-ARCH의 예외 처리 정책이 T4-TASK에 명시되어 있는가? → 없으면 **[중간]**

### 체크 2 — T4 내부 일관성
- [ ] 같은 API를 두 개 이상의 TASK에서 각각 구현하도록 명세되어 있는가? → 중복 발견 시 **[중간]**
- [ ] 같은 데이터/상태를 두 TASK에서 서로 다른 방식으로 처리하도록 명세되어 있는가? → **[중간]**
- [ ] 같은 Props/파라미터 이름이 T4 전체에서 일관되게 사용되고 있는가? → 불일치 시 **[중간]**
- [ ] 파일 경로 기준이 T4 전체에서 일관되게 사용되고 있는가? (예: 일부 `components/`, 일부 페이지 루트) → **[낮음]**

### 체크 3 — 초기 진입 시나리오
- [ ] WS/SSE를 사용하는 컴포넌트에 초기 마운트 시 데이터 로드 방법이 명세되어 있는가? → 없으면 **[높음]** (WS/SSE만으로는 페이지 첫 로드 시 빈 화면)
- [ ] 페이지 첫 로드 시 사용자가 보는 초기 화면 상태가 명세에 있는가? → 없으면 **[중간]**

### 체크 4 — 기존 코드 참조
- [ ] "기존 방식과 동일", "기존 패턴 참조" 류 표현이 있는 TASK에 참조 대상 파일 경로가 명시되어 있는가? → 없으면 **[중간]**
- [ ] 위 표현이 있는 TASK에 참조 대상 메서드명이 명시되어 있는가? → 없으면 **[중간]**
- [ ] 위 표현이 있는 TASK에 "직접 참조 후 재사용" 강제 문구가 있는가? → 없으면 **[중간]**

### 체크 5 — 백엔드 명세 검증
- [ ] Repository 쿼리에서 symbol 외 market_type 등 필수 필터가 누락되지 않았는가? → 누락 시 **[높음]** (데이터 혼재 가능)
- [ ] Service 간 의존성이 메서드 파라미터로 전달되는 방식으로 명세되어 있는가? → 있으면 **[중간]** (Spring DI 원칙 위반, 생성자 주입으로 명세할 것)
- [ ] 폴링 fallback이 명세되어 있으면 해당 REST 엔드포인트가 T4 내에 구현 TASK로 존재하는가? → 없으면 **[높음]**

### 체크 6 — 프론트엔드 명세 검증
- [ ] React setState로 동일값을 세팅하여 리렌더를 기대하는 명세가 있는가? → 있으면 **[높음]** (동일값 setState는 리렌더 안 됨)
- [ ] useEffect 의존성 배열이 명세에서 암묵적으로 잘못 설정될 가능성이 있는가? → **[중간]**
- [ ] 화면에 표시되는 텍스트/레이블이 실제 기술 용어와 다르게 명세되어 있는가? (예: Delta → 거래량) → **[중간]**

### 판정 및 후속 처리
- 높음/중간 있으면 → 반려. 코더에게 검토사항 전달
- 높음/중간 없으면 → 승인
  - "T4-TASK.md 검수가 완료되었습니다. 코딩작업 승인." 출력
  - 프론트 TASK 확인 후 DESIGN 필요 여부 판단:
    - **DESIGN 필요** (아래 중 하나): 신규 컴포넌트 추가 / 기존 레이아웃 구조 변경 / 상태 3개 이상 신규 추가
      → "T9-DESIGN 작성이 필요합니다. 디자이너에게 전달 후 [2.5단계] 진행"
    - **DESIGN 불필요** (버튼/텍스트 단순 추가, 스타일 변경, 기존 컴포넌트 내부 수정)
      → [3단계]로 이동
