# Agent Runbook

## 목적
- AI 작업 절차를 고정해 토큰 사용과 시행착오를 줄인다.

## 기본 절차
1. `chs-rules.md` 확인
2. `docs/agent-entry.md`의 TOP 파일 우선 확인
3. 필요한 최소 범위만 탐색
4. 수정 전 간단 계획 공유
5. 코드 수정
6. 검증 명령은 실행하지 않고 사용자에게 제공
7. 사용자 로그 수신 후 추가 수정

## 탐색 규칙
- 대규모 탐색 전 사용자 승인 필수
- 미승인 시 후보 파일 1~3개와 필요한 라인만 확인

## 검증 규칙
- lint/test/build 자동 실행 금지
- 수정 후 검증 명령어만 제공

## 완료 보고 템플릿
1. 미실행 검증 항목
2. 로컬 실행 명령
3. 수동 확인 포인트(페이지/버튼/시나리오)

## 로그 기반 수정 템플릿
- 입력: 사용자 제공 에러 로그
- 처리: 원인 파일/라인 특정 -> 최소 수정 제안/적용
- 출력: 변경 파일 + 재검증 명령어
