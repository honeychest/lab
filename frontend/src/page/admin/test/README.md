# Admin / Test (`/admin/test`)

운영자가 **브라우저에서 직접** 상태를 확인하거나 디버깅할 때 쓰는 화면이다.  
**자동 테스트·CI·회귀 검증을 대체하지 않는다.**

## 실행(Write) vs 관찰(Read)

- **실행** — 로그인, 갱신, 주문, 설정 변경 등 **부작용이 있는 동작**은 **기존 프로덕션 API**를 호출한다. 이 영역에만 비즈니스 로직을 새로 두지 않는다.
- **관찰** — “지금 토큰·쿠키·캐시가 어떤가”처럼 **상태만 보는 요청**은 `/api/admin/test/<도메인>/debug/...` 형태의 **디버그 전용 엔드포인트**로 둔다. 디버그 API는 **기존 서비스 조회·요약**만 하고, 규칙을 컨트롤러에 복제하지 않는다.

## API 래퍼 (`src/api/adminTest/`)

`/admin/test` 화면에서만 import 한다. 엔드포인트 문자열은 도메인별 파일에만 둔다.

- `adminTest/auth.js` — 로그인(실행), 쿠키/토큰 디버그(관찰)

## shared (공통 부품)

`/admin/test` 아래에서만 쓰는 UI·유틸이다. 다른 페이지에서 import 하지 않는다.

- `shared/logApiCall.js` — API 호출 한 번을 감싸 시각·소요·상태·본문·에러를 기록
- `shared/ApiCallLogPanel.jsx` — 위 기록을 같은 형식으로 표시

## 탭(도메인)

탭 이름은 **기능 종류**(예: “디버그”, “도구”)가 아니라 **비즈니스 도메인**(Auth, Trade, Monitor, User 등)을 기준으로 나눈다.

## 백엔드

`admin.test.<도메인>` 쪽 컨트롤러는 **서비스 위임**만 한다.

## 보안

접근 제어·권한은 **백엔드가 최종 방어선**이다. 프론트 가드는 UX용이다.
