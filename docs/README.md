# 저장소 문서 정책

팀과 공유할 문서와 로컬 전용 파일을 구분하기 위한 기준입니다. 실제 무시 목록은 루트 `.gitignore`가 최종입니다.

## 커밋·공유해도 되는 경로

| 경로 | 용도 |
|------|------|
| 루트 `README.md` | 프로젝트 개요 |
| `docs/` (이 디렉터리) | 설계 메모, 온보딩, 팀 공유 문서 |
| `springboot/docs/` | 백엔드 관련 기술 문서 |
| `frontend/docs/` | 프론트 목업·문서 |
| (선택) 루트 `TODOS.md` | 공개해도 되는 작업 목록 |
| (선택) 루트 `ARCHITECTURE.md` | 아키텍처 개요 |

위 파일들은 `.gitignore`에 넣지 않습니다. 민감한 내용은 커밋하지 않거나 별도 비공개 저장소를 사용하세요.

## 로컬·비공유(`.gitignore` 유지)

- `CLAUDE.md` — 에이전트/개인 워크플로우 메모
- `.cursor/`, `.idea/`, 에디터 로컬 설정
- `.claude/settings.local.json` 등 `*.local.json` 패턴이 있는 항목
- `/devcontext/`, `/logs/` 등 런타임·실험 산출물

로컬 전용 문서를 루트에 두고 싶다면, 위 목록처럼 `.gitignore`에 파일명을 추가하는 방식이 맞습니다.

## 새 문서를 추가할 때

1. 팀과 공유할 내용 → `docs/` 또는 모듈별 `*/docs/` 아래에 두고 커밋합니다.
2. 기계 생성물·캐시 → `.gitignore`에 패턴을 추가합니다 (예: Python은 `__pycache__/`, `*.pyc`).
